Global Scope vs Local Scope

Global Scope: The variables defined outside of any function or curly brackets are known as global variables and have global scope. Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable.

Local Scope: If you were to define some variables inside curly brackets {} or inside a specific function then those variables are called local variables The local variables have a very confined scope which is called the local scope But, with the release of ES6, the local scope was further broken down into two different scopes

  • Function scope
  • Block scope

Block Scope: Block scope is also a sub-type of local scope. The block scope can be defined as the scope of the variables inside the curly brackets {}. Now, these curly brackets can be of loops, or conditional statements, or something else. You are only allowed to refer to these variables from within the curly brackets {}.

Event Loop

JavaScript is a single-threaded synchronous programming language.Single-threaded means that the main thread where JavaScript code is run, runs in one line at a time manner and there is no possibility of running code in parallel.When something asynchronus comes in the function execution stack it automatically passes into the WEB API. WEB API then executes that and passes the event/callback function to the queue. An event loop is something that pulls stuff out of the queue and places it onto the function execution stack whenever the function stack becomes empty.This cycle is called the event loop and this how JavaScript manages its events.

Undefined in JavaScript

The global undefined property represents the primitive value undefined. It is one of JavaScript's primitive types.We can get undefined value in many ways:

  • variable that is not assigned will give undefined
  • function with no return value
  • in function if any parameter that is not passed will be undefined
  • property that doesn't exists on an object will give undefined
  • accesing array elements outside of the index range will give undefined
  • deleting an element inside an array
  • setting a value directly to undefined
